# Translation Namespace Files

This directory contains the split Hebrew translation files for efficient loading.

## Files:

### common.json (1184 keys, 43 KB)
**Always loaded on every page**
- Short UI labels (buttons, forms, navigation)
- Generic actions (save, delete, cancel, etc.)
- Common terms used across the application

### editor.json (157 keys, 20 KB)
**Loaded only on editor page**
- Editor-specific terminology
- Line/region/mask operations
- Keyboard shortcuts
- Transcription panel

### other.json (510 keys, 66 KB)
**Lazy loaded as needed**
- Document management
- Model training
- Image operations
- User/team management
- All other translations

## Performance Impact:

### Before (monolithic he.json):
- Every page loads: **162 KB**

### After (split files):
- Homepage: **43 KB** (common only) → **67% smaller!**
- Editor: **64 KB** (common + editor) → **51% smaller!**
- Other pages: **109 KB** (common + other) → **16% smaller!**

## Usage:

```javascript
// In main.js or i18n setup
import heCommon from '@/locales/he_split/common.json'

const i18n = createI18n({
  messages: {
    he: heCommon  // Load common by default
  }
})

// On editor page (lazy load)
const heEditor = await import('@/locales/he_split/editor.json')
i18n.global.mergeLocaleMessage('he', heEditor.default)

// On other pages (lazy load)
const heOther = await import('@/locales/he_split/other.json')
i18n.global.mergeLocaleMessage('he', heOther.default)
```
